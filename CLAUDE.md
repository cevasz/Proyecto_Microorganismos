# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

**NeuroColony-EC** — a multi-agent simulator of *E. coli* colonies that couples three ML models to biophysical spatial dynamics:

- **RL (`modules/rl_agent`)** — PPO policy for run-and-tumble chemotaxis.
- **Neural ODE (`modules/neural_ode`)** — Physics-Informed Neural ODE (PI-NODE) for internal CheY-P/ppGpp signaling.
- **LSTM (`modules/lstm_division`)** — binary classifier predicting cell division from metabolic history.

The codebase, comments, and CLI output are in **Spanish**. Keep new code/comments in Spanish to match.

## Commands

Everything is run as a Python module from the repo root (imports use absolute `from env...`, `from modules...` paths). Activate `venv/` first.

```bash
# Run the simulation
python -m simulation.runner --agents 50 --steps 100000 --render True --model models/ppo_chemotaxis_final.zip
#   --dynamic-env True (default) uses the real FDM diffusion env; False uses MockEnvFields
#   --model is optional; without it the RL policy is random run/tumble

# Train each module (each writes to models/*.pt or models/*.zip)
python -m modules.rl_agent.train --timesteps 2000000      # -> models/ppo_chemotaxis_final.zip
python -m modules.neural_ode.trainer                       # -> models/neural_ode_best.pt
python -m modules.lstm_division.trainer                    # -> models/lstm_division_best.pt

# Statistical validation vs. biology (needs models/ppo_chemotaxis_final.zip)
python -m validation.run_length_dist                       # -> validation/results/

# Serialize runs to JSON scenarios (portfolio export)
python -m simulation.export_run

# Tests
pytest                                                     # full suite
pytest tests/test_colony.py                                # one file
pytest tests/test_colony.py::test_colony_step              # one test

# Format
black . && isort .
```

## Architecture

### Vectorized colony, lightweight agents
The load-bearing design decision (a recent refactor): `Bacterium` (`agents/bacterium.py`) is a **passive state holder** — position, energy, `internal_state` (the 4-dim ODE vector), and history buffers. It runs **no** neural inference itself.

All model inference happens in **batch** inside `Colony.step()` (`simulation/colony.py`), which owns single shared instances of `PINode` and `DivisionLSTM`. Each step:
1. Stacks every agent's `internal_state` into `[N,4]` and integrates the Neural ODE once.
2. Builds observations and calls the RL policy once for all agents.
3. Calls `agent.update_kinematics_and_metabolism()` per agent (physics only).
4. Batches division prediction through the LSTM for eligible agents (those with a full 60-sample `history`).
5. Handles births/deaths in a single O(N) list rebuild (no `list.remove` in a loop).

When adding per-agent behavior, follow this pattern: store state on `Bacterium`, do tensor work in `Colony`.

### Two environment implementations — watch the mismatch
- `env/grid.py` `EnvironmentFields` — the real environment. Wraps two `DiffusionGrid` FDM solvers (`env/diffusion.py`) for glucose and oxygen with CFL sub-stepping. **Nutrient source is the left edge, center at `(0, ny*dx/2)` ≈ `(0, 50)`.** `get_observation` returns 4 values: `[glucose, grad_g, oxygen, grad_o]`.
- `simulation/runner.py` `MockEnvFields` — a static radial gradient with a finite nutrient reserve. **Center at `(50, 50)`.** Used when `--dynamic-env False`.

`Colony.__init__` branches its initial spawn placement on `env_fields.center[0]` (0.0 → left-edge spawn, else radial). Any new env must expose `center`, `get_observation(x,y)`, `consume(x,y,amount)`, and `step()`.

### RL observation-shape gotcha
`env/environment.py` `BacteriumEnv` is a **separate, simplified Gym env used only for PPO training** — 4-dim obs `[pos_x, pos_y, grad_x, grad_y]`, dummy gradient toward `(100,100)`, action `[run_duration, tumble_probability]` in `[-1,1]`. It does **not** use `EnvironmentFields`.

Because of this, `models/ppo_chemotaxis_final.zip` expects a **4-dim** obs, but `Colony` naturally builds a **6-dim** obs (`[env(4) + internal(2)]`). `Colony.step()` detects `rl_policy.observation_space.shape[0]` and, if it's 4, reconstructs a synthetic 4-dim obs with a gradient pointed at `(50,50)`. If you retrain the policy on a 6-dim space, this branch is bypassed. Keep the two obs definitions in sync when changing either.

### Model weights & data caveats
- `.gitignore` excludes `*.pt`/`*.pth`, so **only `models/ppo_chemotaxis_final.zip` is committed**. `models/lstm_division_best.pt` and `models/neural_ode_best.pt` are untracked and must be regenerated by training. `Colony` loads the LSTM weights if present and falls back to random init with a warning otherwise; it does **not** load Neural ODE weights at all (uses fresh `PINode`).
- Despite README/INSTRUCCIONS references to NCBI GEO GSE4513 and Taheri-Araghi 2015, **both training datasets are synthetic**, generated in-code (`modules/neural_ode/dataset.py::_generate_synthetic_data`, `modules/lstm_division/dataset.py::_generate_synthetic_adder_data`). There are no real data files.

### PI-NODE physics loss
`PINode.compute_loss` (`modules/neural_ode/pi_node.py`) adds three physics penalties to the MSE: mass conservation (`sum(dz/dt)≈0`), non-negativity, and a Michaelis-Menten upper bound on `dCheY-P/dt`. This is the ablation target ("PI-NODE vs. plain Neural ODE") called out in INSTRUCCIONS.md.

## Config & logging
- `configs/default.yaml` centralizes all biological/physical/hyperparameters. `train_ppo_agent` reads `rl.learning_rate` and `rl.gamma`; most other values are currently hardcoded at their use sites rather than read from the YAML — check the call site before assuming a config value is wired in.
- `simulation/runner.py` writes `runs/run_<timestamp>/metrics.jsonl` (one JSON object every 100 steps, including full agent positions and the glucose field). These accumulate and are gitignored-adjacent clutter — the `runs/` tree is not ignored, so avoid committing new run dirs.

## Portfolio export & static viewer (`portfolio_deployment/`)
A self-contained pipeline that turns simulation output into a static, no-backend HTML viewer for hosting (e.g. a Hugging Face Static Space):
- `generate_data.py` runs `Colony` (via `simulation.export_run`-style loop) and serializes agent positions, orientation, energy, and a subsampled 25x25 glucose grid per frame into `scenario_ppo.json` / `scenario_extinction.json`.
- `generate_cell.py` produces `cell_internal.json`, single-cell telemetry (energy/ATP, transcription, translation, P(division)) driven by the real trained LSTM (`models/lstm_division_best.pt`).
- `build_index.py` embeds all three JSON files directly into `viewer.html` to produce a self-contained `index.html` (no fetch/CORS, opens via double-click).
- Regenerate with: `python portfolio_deployment/generate_cell.py && cd portfolio_deployment && python build_index.py`.
- See `portfolio_deployment/DEPLOY.md` for hosting steps. Treat `viewer.html` as the editable template and `index.html` as a build artifact.
